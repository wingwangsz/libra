# Part A — 自动升级

## 1. 目标与结论

为官方安装提供可选自动升级（`upgrade.mode=auto`）。在本计划限定的平台、信任引导和回滚约束下**可行**；未满足这些前置条件的安装只能使用 `manual`/`off`，不得静默升级为“官方安装”。

**首期支持矩阵**：Linux x86_64、Linux aarch64、macOS aarch64。Windows x86_64 继续发布但首期自动升级明确返回 `UnsupportedPlatform` 并保持原二进制；Windows 需独立 helper/`ReplaceFile` 设计通过后才能启用。macOS x86_64 当前不在 release matrix，安装器和 manifest 均不得宣称支持。

## 2. 安装目录与官方判定

```text
exe = canonical(current_exe())           // 正常已安装模式必须名为 libra
INSTALL_DIR = parent(exe)
MARKER = INSTALL_DIR/.libra-official-install.json
```

专用入口有互斥模式：`version/pre-install/bootstrap` 只允许从 `INSTALL_DIR/.dl-<random>` 启动，验证父目录 fd、candidate inode 与父进程 fd 相同、no-follow regular、名称匹配 `.dl-[A-Za-z0-9]{16,}`；`post-install` 只允许 basename=`libra`，并验证 current_exe inode 与锁内固定 target fd 相同。任一模式错配均拒绝。普通命令路径仍只允许已安装目标名 `libra`。

官方：MARKER 有效、`marker.version`/`marker.sha256` 匹配 TARGET、`marker.install_source=official_signed_manifest`，且 `INSTALL_DIR` 通过 §A.5 所有权/权限/no-follow 校验。仅“文件位于默认目录”或“当前二进制给自己计算 hash”均**不能**建立官方来源。

存量 bootstrap 仅允许默认 `~/.libra/bin/libra`，且必须重新获取并用**安装器内预置公钥**验证与当前 `(version, platform, sha256, size)` 完全一致的签名 manifest；验证工具不可用、manifest 过期或当前版本已不在受信发布记录中时保持无 marker、warning 一次并视为 `off`。`LIBRA_BASE_URL`、测试 endpoint、手工复制和第三方包管理器安装均不得写官方 marker；包管理器若需接入，必须由各自签名/receipt 适配器单独定义。

## 3. 配置 `upgrade.mode`

文件：`{LIBRA_HOME}/upgrade/settings.json`。新增 Rust 侧唯一 `resolve_libra_home()`，与 `install.sh` 的 `LIBRA_HOME`/`HOME` 规则及 `LIBRA_CONFIG_GLOBAL_DB` 测试隔离契约一致；升级配置是保留命名空间，不得同时落入 SQLite config。

| 操作 | 行为 |
|---|---|
| `set --global upgrade.mode V` | 仅接受 `auto`/`off`/`manual`（大小写不敏感）；非法值 CLI 报错；原子写 `mode` |
| `set`（无 global） | 拒绝 |
| `get --global` | 读 `mode`；缺失→`off` |
| `unset --global` | 写 `mode=off`（**保留文件**，不删除） |
| `list --global --show-origin` | `file:{path}` |

读取时 `mode` 非三枚举之一→get 报错；文件损坏→get 错；升级路径未知/损坏均视 `off` 并 warning（一次）。

所有可到达同一 key 的 config 拼写（legacy action、`--get-all`、`--get-regexp`、`--add`、类型转换、JSON）必须经过同一 reserved-namespace router：只允许上述单值操作；local/system/multivalue 操作 fail-closed。`list` 不得再从 SQLite 输出第二份 `upgrade.mode`。

## 4. Marker

```json
{ "schema_version":1, "installed_at":"<RFC3339>", "install_source":"official_signed_manifest", "platform":"<enum>", "version":"<SemVer>", "sha256":"<64hex>", "size":123, "manifest_key_id":"<id>" }
```

`install.sh`：先用脚本内预置公钥和系统受支持的 Ed25519 verifier 验签，再把 artifact 写入 `INSTALL_DIR` 内同文件系统的唯一临时文件并验证 size/hash。已有支持新协议的 Libra 时调用其隐藏命令完成 no-follow 校验、锁、替换和 marker；**fresh install 或 legacy binary 不支持隐藏命令时**，仅在独立验签已建立候选信任后调用候选的严格入口 `__upgrade-bootstrap-install --signed-envelope-fd …`。候选必须再次用内置 trust table 验 envelope/hash，入口只允许安装自身已打开的 inode 到已验证目录，不接受任意 source/target/command；同样执行 §A.5/§A.7 的锁、fsync、backup 和 marker 协议。不能独立验签时可按现有手工安装路径完成安装，但不得生成官方 marker，也不得默认开启 auto。禁止让未通过独立签名链的下载物为自己背书。

## 5. 锁

`libra __upgrade-lock exec --install-dir <canonical_dir> -- <cmd>`：Unix 使用 advisory file lock；校验目录绝对路径、owner 为当前 euid、非 group/world-writable（sticky 私有目录例外须单列）、目标与状态文件均为 regular file 且 no-follow。打开并持有目录 fd，后续 target/lock/marker/txn/retry 操作全部相对该 fd，防目录替换与 symlink TOCTOU。文件权限：状态 `0600`、候选/target `0755`、目录 `0700`。升级用 `try_lock`（忙=Skip）。隐藏命令跳过升级检查。Windows 首期不进入该路径。

## 6. Manifest

**端点**：`https://download.libra.tools/libra/releases/stable/manifest-v1.json`

**信任表**（`trusted_keys.rs`）：`key_id`、`ed25519_pubkey`、`not_before`、`not_after`、`generation`，以及客户端版本编译期固定的 `MIN_TRUSTED_KEY_GENERATION`。签名数组拒绝重复 `key_id`；有效签名 generation 必须 `>= max(manifest.min_key_generation, MIN_TRUSTED_KEY_GENERATION)`，manifest 和墙钟都不能降低/提前改变水位。先做纯密码学验签，再要求签名 key 的有效区间覆盖签名 payload 的 `published_at`，且 `expires_at <= key.not_after`。轮换 overlap 通过旧客户端继续携带较低编译期 floor、新客户端发布时提高 floor 实现；不依赖本地/HTTP 时间触发 generation。

**签名 job 隔离**：私钥只能进入独立、最小权限、protected-environment/KMS 签名 job；该 job 只消费已完成构建的不可变 hash/size 元数据。所有 actions 固定 commit SHA；secret 可见后禁止下载/执行 rclone 安装器、Homebrew、Chocolatey 或其它网络脚本；build/upload matrix 永不接触任一签名私钥，轮换时也不得把两把私钥暴露给构建 job。

**URL 解析**：结构化 URL 解析后校验：scheme=https、host=download.libra.tools、port 空或 443、path 四段 `/libra/releases/v{tag}/libra-{platform}`（Unix 无后缀；Windows `.exe` 后缀为 R0 后项，首期 auto-upgrade 明确返回 `UnsupportedPlatform` 不进入下载路径）、query/fragment 为空、tag 为 release SemVer；并强制 `tag == payload.version`、URL `platform == artifact.platform`、artifact platform 唯一且覆盖精确 release matrix。所有跨字段校验在任何 anti-rollback/control/time state 写入前完成。禁重定向。

**信封**：`schema_version`、`payload`（base64）、`signatures[]`。验签字节：`b"libra-upgrade-manifest-v1\0" || payload_bytes`。

**payload**：`channel=stable`；`version`（无 prerelease/build）；`control_revision:u64`；`published_at`；`expires_at`；`min_key_generation`；`paused`；`revoked_versions[]`；`artifacts` 与 release matrix 一致的四平台 `{platform,url,sha256,size}`。每次新版本、续签、暂停/恢复或撤回变更都必须严格递增 `control_revision`。客户端只选择自身精确 platform；首期 Windows 选择后返回 unsupported，不下载。`paused=true` 禁止任何新下载/安装；目标版本在 `revoked_versions` 中时，即使 retry/cache 命中也禁止安装。

**时间**：HTTPS `Date` 先仅作本轮 provisional 值；必须满足 `published_at-300s <= https_date < expires_at`，且 envelope 完成密码学验签、key/payload/URL/size/control 全量校验后，才与 anti-rollback state **同一原子写**推进 `trusted_time_floor=max(old_floor,published_at,https_date)`。错误状态码、无效 envelope 或异常未来 Date 绝不写 state。expiry 判断用 `effective_now=max(local_wall_clock, trusted_time_floor, https_date)`；本地未来时间只会拒绝当轮，不写 floor，修正后可恢复。本地低于 floor 超 300s 时禁止 cache 安装并强制在线刷新。测试覆盖无效 envelope + 未来 Date 不毒化 state、跨重启回拨和未来墙钟恢复。

**体积**：manifest ≤1MiB；artifact `size` 必须满足 `0 < size <= 128MiB`（**在持久化 `max_seen` 之前**校验；超限拒 manifest 且不写 state）。Stable job 发布前同样校验。

**缓存/节流**：成功接受 manifest 后，以 validated HTTPS Date 持久化 `next_success_check_not_before = https_date + 15min + deterministic_jitter(0..120s)`；正常时钟下跨进程可在此冷却期直接 Skip，故成功联网时暂停/撤回的最大检查传播延迟为 17 分钟。若本地墙钟不在 `[trusted_time_floor-300s, trusted_time_floor+24h]`、冷却早于 floor 或超过 `floor+17min`，忽略冷却并联网。冷却只能跳过检查：任何 artifact 新下载、复用候选或安装，都必须在同一进程先在线接受不低于当前 control revision 的 manifest。失败 backoff 最大 1 小时并使用同样可信上界；`last_good_envelope` 仅诊断，不跨进程授权安装。

**HTTP client**：专用 reqwest client，`https_only(true)`、rustls 主机名/证书校验、`redirect::Policy::none()`、connect/header/body 总 deadline；收到 3xx 即失败，并在读 body 前复核 effective URL。测试覆盖 HTTP downgrade、非 443、query/fragment、redirect loop、TLS hostname、stalled header/body、chunked overflow 和错误 `Content-Length`。

**下载**：流式读取；若 `Content-Length` 存在且 `>size` 或 `>128MiB` 立即中止；逐块计数，超过 `size` 立即中止；结束字节数**必须等于** `size` 且 sha256 匹配。

**测试**：`test-upgrade` feature + `LIBRA_TEST=1`；release 禁用。

## 7. 版本与 Phase

- anti-rollback/control state：持久化 `max_seen`、每平台 artifact identity、`max_control_revision`、该 revision 的 envelope digest、`trusted_time_floor`。版本 `<` 拒；同版本只允许 artifact identity 相同。control revision `<` 一律拒；`==` 仅接受 envelope digest 完全相同；`>` 才接受续签/紧急控制。该状态在 manifest 全量语义校验、平台选择、size、签名和时间策略通过后于锁内 durable 写入。测试必须覆盖“先接受撤回，再收到仍未过期的旧未撤回 envelope”。
- 下载条件：`manifest.version > installed_target_version`；候选 `--version` 必须等于 manifest.version。
- Phase A（5s manifest + 10s 下载软预算）：下载到 `${INSTALL_DIR}/.dl-*`（同文件系统；`TempGuard` RAII）→流式 sha256+精确 size→`sync_all`→chmod 0755→以候选执行专用隐藏入口 `__upgrade-probe --kind version|pre-install --expected-version …`。该入口在 argv 解析最前端识别，只执行无副作用自检，**明确绕过自动升级、txn 恢复、repo preflight、配置写入和后台任务**，不能转发任意用户命令。各 probe 有独立硬超时；子进程启用 kill-on-drop，并以进程组/job object 终止所有后代后 `wait` 回收。timeout/cancel 必须中止 body stream、回收进程并清理，禁止 detached 任务。
- Phase B：txn 显式记录 `old_target: Absent | Present { hash, marker_snapshot }`。Present 分支先 durable backup 再原子覆盖；Absent 分支不要求 backup。覆盖后执行固定目标模式的 post-probe；成功后 durable 写 `PostProbePassed`、marker/state、`Committed`。写 `Committed` 后核验 target/marker/state identity，删除 backup/candidate并 fsync目录，再删除 txn并再次 fsync；txn 必须最后删除。probe 失败按 Present 回滚、Absent 删除 new target。恢复失败保留材料并返回 FatalRecovery。

**Txn 恢复**（输出模式解析后、仓库 preflight 前、marker 检查前）：先取得升级独占锁并持有至最后 fsync。状态机为 `Prepared`、`BackupDurable`、`CandidateInstalled`、`PostProbePassed`、`RollbackIntent`、`AbortAbsentIntent`、`Committed`。probe 失败后在修改 target **之前** durable 写 intent：Absent 写 `AbortAbsentIntent` 后才删除 new target；Present 写 `RollbackIntent` 后才以 backup 原子恢复 old target。intent 恢复同时接受修改前/后 identity，幂等完成 target 修改、清理残留、fsync，最后删 txn。`CandidateInstalled` 必须重新 post-probe；`PostProbePassed` 继续提交；`Committed` 幂等提交清理。只有 identity 不匹配已声明状态时 FatalRecovery。

恢复决策表（所有存在文件先做 owner/regular/no-follow/hash 校验；其它组合 FatalRecovery）：

| txn state | old_target | 实测 identity | 动作 |
|---|---|---|---|
| Prepared | Absent | target absent + candidate=new | 删除 candidate/txn，fsync，恢复未安装 |
| Prepared | Absent | target=new + candidate absent | 说明 rename 已完成但状态未落盘；写 CandidateInstalled 后执行 post-probe |
| Prepared | Present | target=old + candidate=new + backup absent | 删除 candidate/txn，保持 old |
| Prepared | Present | target=old + candidate=new + backup=old | 补写 BackupDurable 后继续原子覆盖 |
| BackupDurable | Present | target=old + candidate=new + backup=old | 继续原子覆盖并写 CandidateInstalled |
| BackupDurable | Present | target=new + candidate absent + backup=old | 补写 CandidateInstalled 后执行 post-probe |
| CandidateInstalled | Absent/Present | target=new；Present 时 backup=old | 重新 post-probe；成功→PostProbePassed，失败→按分支删除/回滚 |
| PostProbePassed | Absent/Present | target=new；marker/state 可旧或新 | 幂等写 marker/state 后写 Committed |
| AbortAbsentIntent | Absent | target=new 或 absent；candidate 可有/无 | 若 target=new 则删除；幂等删 candidate，fsync，最后删 txn |
| RollbackIntent | Present | target=new + backup=old，或 target=old + backup 可有/无 | 若 target=new 则原子恢复 backup；核对 target=old 后清残留、fsync，最后删 txn |
| Committed | Absent/Present | target/marker/state=new | 幂等清理 backup/candidate，最后删除 txn |

## 8. 输出

human 失败 warning；json/machine 无中途 stderr；`--exit-code-on-warning` 仅末尾单 envelope（stdout 保留命令 JSON）。

## 9. 发布

- Per-tag：所有平台先完成构建与 hash/size 汇总，再由单一 publish job 使用 storage `If-None-Match:*` 条件创建；禁止 prerelease tag；同 tag 已存在对象逐一核对 identity，不符 fail，禁止覆盖。只有全部必需 artifact 可读且 identity 正确后才允许签 stable manifest。
- Stable job：读当前并验签；`new < current` fail；`new > current` 且双方为 release SemVer。普通新版本发布必须从当前 payload 逐字节继承 `paused`/`revoked_versions`；任何控制字段变化只能走 protected 紧急 job。新发布 `expires_at=published_at+90d`；按轮换策略签名；对象存储使用 ETag/version CAS。
- 续签（`new == current`）：除 `control_revision`、`published_at`、`expires_at`、`signatures` 外，`version`/`artifacts`/`channel`/`min_key_generation`/**`paused`/`revoked_versions` 必须逐字节不变**；时间与 revision 单调增大。普通 renew job 不得恢复/撤回紧急控制。
- 每周 renew job 执行续签分支。
- 紧急控制：独立 protected job 可只修改 `paused`/`revoked_versions` 并提升 `published_at`，仍须满足当前 key-generation 签名与 CAS。恢复发布需更高 `published_at` 且显式审计记录。客户端若已运行撤回版本，不自动降级；每次成功检查输出高优先级 warning 和升级到更高修复版本的指引。测试锁定旧 cache 不得绕过撤回。

## 10. CLI

```rust
let output = OutputConfig::resolve_from_cli_args(&args)?;
match tokio::time::timeout(UPGRADE_BUDGET, upgrade::phase_a(&args, &output)).await {
    Ok(Ok(PhaseAOutcome::Ready(job))) => {
        match upgrade::phase_b(job, &output).await {
            Ok(PhaseBOutcome::Installed) => {}
            Ok(PhaseBOutcome::RolledBackWarning(w)) => upgrade::report_upgrade_outcome(w, &output),
            Err(PhaseBError::FatalRecovery(e)) => return Err(e),
        }
    }
    Ok(Ok(PhaseAOutcome::Skip)) => {}
    Ok(Err(e)) => upgrade::report_upgrade_outcome(e, &output),
    Err(_) => upgrade::report_upgrade_outcome(Outcome::Timeout, &output),
}
```

`FatalRecovery`、启动时 txn 损坏、回滚失败和标准 target hash 无法归类时，必须在仓库 preflight/用户命令 dispatch **之前**退出；只有已确认 target 恢复到旧 hash 的 `RolledBackWarning` 才能继续原命令。测试用 side-effect sentinel 断言 fatal 路径未执行 `commit`/`add` 等用户命令。

## B.3. 数据流（修订后，可实施）

**原则**：rename 检测全程使用 **repo-relative** 路径键；`to_relative()` 仅在 pathspec 过滤之后、人类展示之前。

```text
collect_status_data(args)
  ├─ rename_threshold ← parse_status_rename_threshold(args)   // §B.5；缓存模式恒为 None
  ├─ staged_repo ← changes_to_be_committed_safe()             // 已是 repo-relative
  ├─ worktree_scan ← scan_workdir_display()                   // §B.3.3：主扫描亦产 partial/IoBlocked
  ├─ unstaged_repo ← build_unstaged_changes(index, worktree_scan)
  ├─ rename_untracked ← status.renameUntracked（默认 false）
  ├─ eligible_deleted ← pathspec_filter_sources(unstaged.deleted, pathspec_set)
  ├─ if rename_threshold.is_none() || eligible_deleted.is_empty() → skip probe
  ├─ else if !rename_untracked → skip probe（默认 Git：未跟踪不作 destination）
  ├─ else probe_roots ← pathspec_probe_roots(repo_root, pathspec_set)   // §B.3.1.1
  │     rename_probe ← BoundedWorktreeWalker::probe(probe_roots, &mut ProbeBudget)  // §B.3.2
  ├─ merge_probe(outcomes) → rename_destinations_raw         // §B.3.2（复合 Result）
  │     • destinations = 各 root.partial/paths 并集
  │     • 任一 root.io_blocked 非空 → 传播至 StatusData.io_blocked
  │     • 任一 root.truncated → Truncated warning（仍保留 destinations）
  ├─ rename_destinations ← filter_rename_destinations(       // §B.3.1.2 资格
  │       rename_destinations_raw, index, ignore, pathspec_set)
  ├─ pathspec_filter_each_end(staged, unstaged_tracked)   // 各端独立过滤；old-only=D，new-only=A
  ├─ detect(staged + unstaged_tracked + rename_destinations)
  ├─ collapse_untracked_markers(consumed_destinations, probe_completeness)  // §B.3.5
  ├─ build StatusRecords（repo-relative）；若 io_blocked 非空 → 见 §B.6.0.1
  ├─ if worktree_scan.io_blocked 非空 OR probe.io_blocked 非空
  │     → **禁止** DirtyCache::replace_all*（保留旧快照；§B.3.3）
  └─ render：
        human/short/porcelain ← 有 IoBlocked 则 fail-closed（§B.6.0.1）
        json/api ← records + renames + io_blocked[]；is_clean 见 §B.6.0.1
```

测试（pathspec 边界）：`pathspec_old_only_new_only_matrix`、`probe_dest_filtered_by_pathspec_exclude`、`probe_dest_icase_root_still_pathspec_filtered`、`rename_untracked_pathspec_no_cross_boundary`。

### B.3.1. 未跟踪文件与 unstaged rename

- unstaged：`deleted`=index 有/worktree 无；`new`=worktree 有且 index 无。
- **Git 默认（R0）**：未跟踪路径**不**作为 rename destination；`mv` tracked→untracked 位置 → `D` + `??`（与 Git 一致）。
- **Libra 扩展（默认关闭）**：`status.renameUntracked=true` 才启动 probe、填充 `rename_destinations` 并允许 unstaged 配对；否则 `unstaged.new` 仅用于 `??` 展示，**不**进入 rename 检测。测试：`probe_skipped_when_untracked_disabled`、`git_vs_libra_untracked_rename_matrix`。
- **主扫描**：保留现有 Normal 折叠；`??` 报告语义与 Git 一致。
- **rename probe**：`rename_threshold.is_some()` **且** `deleted` 非空时启动；`--no-renames`/配置关闭 → **不** probe（`probe_skipped_when_renames_disabled`）。
- **目录预算**耗尽 → `Truncated`（§B.3.2 全局 ProbeBudget）。
- **WorktreeReadBudget** 耗尽 → 跳过需读取的 worktree exact/inexact；HEAD/index Known-OID exact **仍**配对（独立于 ObjectReadBudget）。

#### B.3.1.1. Probe roots（与 `-uno` 解耦）

`pathspec_probe_roots(repo_root, pathspec_set: &PathspecSet) -> Vec<PathBuf>`（**独立**于 `scan_workdir`/`unstaged.new` 展示集；**禁止**对 raw argv 字符串做朴素 `parent()`）：

1. 无 pathspec / `pathspec_set.is_empty()` → `[repo_root]`。
2. **exclude-only**（`!has_positive`，正集合≡全仓库）→ `[repo_root]`（不得因纯 `:!` 漏扫）。
3. 有 positive specs → 对每个 **positive** matcher 推导**保守** probe root：
   - `Prefix`/`Exact`/`Glob` 能解析出确定目录祖先 → 该祖先；
   - `:(icase)` / 大小写折叠 / 通配过宽导致无法在大小写敏感 FS 上安全收窄 → **回退 `repo_root`**（宁可多探，不可漏 destination）。
4. 结果按 repo-relative 规范化后去重，并去掉被另一 root **祖先包含**的冗余子根（`a/` 覆盖 `a/b/`）。
5. `--untracked-files=no` 时仍执行 probe（当 `rename_untracked=true` 且 `deleted` 非空）；probe **永不**向展示层注入 `?`/`??` marker。
6. 测试：`rename_untracked_true_uno_probe_success`、`rename_untracked_true_uno_no_question_mark`、`rename_untracked_true_uno_probe_truncated`、`probe_roots_exclude_only_repo_root`、`probe_roots_icase_falls_back_repo_root`、`probe_roots_dedup_nested_ancestors`。

#### B.3.1.2. Destination 资格（`filter_rename_destinations`）

Probe 产出的路径在进入 `detect` **之前**必须复用主扫描同一分类规则（锚点：`status_untracked.rs` tracked/ignore 分层）。路径**仅当**同时满足才保留为 rename destination：

1. `pathspec_set.matches_path(p)`（roots 保守回退不得越界配对）。
2. **非** index stage-0 tracked（含 case-fold alias：在 `core.ignorecase=true` 时与已跟踪路径大小写折叠冲突 → **排除**，避免 `D a + M b` 误报 `a→b`）。
3. **非** unmerged（任一 stage 1/2/3 存在）。
4. **非** ignore / exclude layer 命中（与 `??` 展示层一致；ignored 不得作 destination）。
5. 是普通文件或 symlink（目录本身不作 destination；gitlink 见 exact 规则）。

测试：`probe_dest_rejects_tracked_path`（`D a` + 修改 tracked `b` → 无 rename）、`probe_dest_rejects_ignored`、`probe_dest_rejects_unmerged`、`probe_dest_rejects_casefold_alias`。

### B.3.2. Probe 归并协议（Normal）

`BoundedWorktreeWalker` 持有**一次 status 调用全局共享**的 `ProbeBudget`：

| 资源 | 上限 | 语义 |
|---|---|---|
| `max_enumerated_entries` | **50_000** | 每次 `read_dir` 取出的 entry 均跨 roots 聚合计入，包含随后被排除者；保证总工作有界 |
| `max_qualified_destinations` | **10_000** | 通过资格后的 destination 另行计入；可独立早于 enumeration 触顶，防候选存储失控 |
| `no_progress_deadline` | 单次阻塞 I/O 10s（测试可注入，不由普通配置关闭） | worker 超时终止；当前路径记 `IoBlocked(io_timeout)` |

对每个 `probe_root` 顺序探测，返回**单一复合结果**（禁止 Truncated/IoBlocked 互斥导致丢状态）：

```rust
struct ProbeRootResult {
    /// 本 root 成功收集的 destinations（含 EACCES/截断前与续扫所得）
    destinations: Vec<PathBuf>,
    /// 预算是否触顶（可与 io_blocked 同时为真）
    truncated: bool,
    /// 本 root 累计的全部 I/O 错误（可多条；稳定排序后的遭遇序或路径序均可，须确定）
    io_blocked: Vec<IoBlockedEvent>,  // { path, reason }
    /// **仅** probe 开始时 root 路径 NotFound
    gone: bool,
}
```

`gone=true` 时 `destinations`/`io_blocked`/`truncated` 均为空/false。`Gone` **不等于** entry race。

**可回收 I/O worker**：基础扫描/probe 使用一个流式 worker，发送 framed `Begin/Record/Checkpoint`；基础 worker timeout 时终止并保留最后 checkpoint 前的 partial，记录 IoBlocked。可选 rename 内容任务（每个 repo OID、worktree path 或 LFS hydration）使用 bounded-concurrency 的**逐任务子进程**，或每任务完成后落 checkpoint 并可重启的 worker；父进程持有稳定排序的 pending queue。单任务 timeout 只标记该 OID/path edge 失败，回收该 process group 后继续剩余 queue，不能终止整个评分批次。worker 入口绕过升级/恢复和写入，只接受匿名 pipe + capability token。仅基础 existence/type/tracked dirty/untracked enumeration 失败映射 IoBlocked；可选读取映射 Metadata/Worktree warning。

Walker **内**循环（**有界枚举**；禁止先无界 `collect` 全目录再计预算）：

- 遍历以 repo-root 目录 fd/handle 为边界并使用 no-follow/beneath 语义；symlink 永远作为叶节点（可作 symlink rename destination），绝不递归。无条件排除实际仓库管理目录（`.libra`，兼容导入场景的 `.git`）、gitlink/submodule 子树和嵌套仓库管理目录；任何解析后逃出 repo root 的 entry 记录 `IoBlocked` 并跳过。
- **资格前置剪枝**：确定 ignored 或不可能匹配 positive pathspec 的目录在打开/枚举子树前剪枝。已从父目录取出的 entry 始终计入 `max_enumerated_entries`，但 pathspec 外/ignored entry 不计 `max_qualified_destinations`，其候选专属 EACCES/timeout 不传播为 IoBlocked。matcher 不能安全判定时保守进入。
- 对每个目录：迭代 `read_dir` 时每取一条 entry 立即消费 `max_enumerated_entries`；资格过滤通过后再消费 `max_qualified_destinations`。任一触顶都设 `truncated=true` 并停止继续枚举/收集；不得为排序继续无界读取。JSON warning 标明触顶的是 `enumeration` 或 `destination` budget。
- **确定性**：完整枚举且未截断的目录按 repo-relative 路径字节序排序后再取 metadata / 递归。目录在 `read_dir` 迭代中触顶时立即停止；OS 枚举顺序无保证，因此截断 partial 的**成员集合不承诺跨文件系统一致**，但已收集结果在输出前稳定排序。禁止为选择“字典序前 50k”继续无界枚举。
- entry `NotFound` TOCTOU → **`continue`**（仍已计预算）。
- entry EACCES/I/O → 追加到 `io_blocked`，**继续** siblings（预算内）。
- root 起始 `NotFound` → `gone=true`。

测试另增：`probe_budget_bounds_wide_directory`、`probe_irrelevant_entries_count_enumeration_not_destinations`、`probe_ignored_subtree_pruned_before_enumeration`，断言两预算分别有界且触顶原因稳定。

**归并（跨全部 `probe_roots`）**：

| 字段 | porcelain / short / human | JSON / API |
|---|---|---|
| `destinations` | 有任一 `io_blocked` 全局非空 → fail-closed；否则纳入配对 | **始终**纳入配对 |
| `truncated` | 可配对 + Truncated warning | 可配对 + `warnings[]` |
| `io_blocked` | fail-closed（§B.6.0.1） | `io_blocked[]` + warning；`is_clean=false` |
| `gone` | 跳过 | 跳过 |

测试：`probe_root_not_found_silent`、`probe_entry_race_skips_only_that_entry`、`probe_dir_eacces_*`、`probe_ioblocked_keeps_prior_sibling_renames_json`、`probe_truncated_partial_still_pairs_rename`、`probe_complete_entries_sorted`、`probe_truncated_resource_bound_without_member_stability`、`probe_never_enters_libra_or_git_metadata`、`probe_directory_symlink_is_leaf`、`probe_excludes_gitlink_and_nested_repo`、`probe_ignored_huge_tree_not_counted`、`probe_pathspec_outside_eacces_not_propagated`、`probe_eacces_then_truncated_keeps_both`（先 EACCES 后触顶：JSON 同时有 `io_blocked[]` + Truncated warning + partial renames；porcelain fail-closed）、`probe_multiple_blocked_paths_reported`。

### B.3.3. 主扫描 partial / IoBlocked（与 probe 同构）

`scan_workdir_display` / tracked dirty 扫描（今日 `status_untracked.rs` 对 `symlink_metadata`/`read_dir` 错误 `?` fail-fast）R0 **必须**改为 accumulator，不得在首个 EACCES 处丢弃已收集条目：

```rust
struct WorktreeScanAccum {
    tracked_dirty: Vec<…>,
    untracked: Vec<…>,
    ignored: Vec<…>,           // `--ignored` / JSON `ignored`；与 status_untracked 一致
    io_blocked: Vec<IoBlockedEvent>,  // path + reason
}
```

协议：

| 事件 | 行为 |
|---|---|
| entry / 子树 EACCES/I/O | 记录 `IoBlockedEvent`；**保留**错误前已收集的 tracked/untracked/**ignored**；继续 sibling（与 probe 续扫同语义） |
| entry `NotFound` TOCTOU | `continue`（不 abort） |

**禁止**对主展示扫描套用 ProbeBudget / Truncated：主扫描无目录预算；仅 rename probe 使用跨 roots 聚合的 `max_enumerated_entries=50_000` 与 `max_qualified_destinations=10_000`。大量 `??` 必须完整报告。

渲染（与 §B.3.2 / §B.6.0.1 一致）：

- **porcelain / short / human**：任一 `io_blocked` → **fail-closed**（`LBR-IO-001`）；不输出半截正文。
- **JSON / API**：输出至错误前已确定的 `staged`/`unstaged`/`ignored`/`renames` + `data.io_blocked[]` + `warnings[]`。

**Dirty cache**：`worktree_scan.io_blocked` **或** probe 聚合 `io_blocked` 非空时，**禁止一切** dirty-cache **内容写**，包括但不限于：
- `DirtyCache::replace_all*` / `replace_all_with_conn`（含 `--scan` / JSON）
- `--check-dirty` 路径的 `prune_and_confirm_with_conn` / 任何 prune、confirm、upsert、delete 条目写

保留既有快照原样；不可读路径**不得**因 `symlink_metadata` 失败被当成「不存在」而 prune NEW 或 confirm DELETED。测试：`ioblocked_scan_does_not_replace_dirty_cache`、`check_dirty_ioblocked_does_not_mutate_cache`（先建含 NEW/DELETED 条目的 cache → 目标路径 chmod 不可读 → `--check-dirty` → 断言条目集合/指纹不变）。

测试：`main_scan_ioblocked_keeps_prior_sibling_json`、`tracked_scan_eacces_partial_json`、`main_scan_ioblocked_keeps_ignored_json`、`ioblocked_scan_does_not_replace_dirty_cache`、`check_dirty_ioblocked_does_not_mutate_cache`。

### B.3.5. Untracked dir marker collapse

`collapse_untracked_markers(consumed, probe_completeness)`（Normal 模式）：

1. 起点：主扫描已有的 `? dir/` / `??` markers（`-uno` 不注入新 marker）。
2. 从 probe/detect **消费**的 untracked destinations 中，按目录聚合「该 dir 下是否仍有**未**配对的合格 untracked」。
3. **Complete**（该 dir 所属 probe root：`!truncated && io_blocked.is_empty()`）且该 dir 下无剩余 untracked → **移除** `? dir/`（避免 `.R old→dir/file` 与 `? dir/` 并存）。
4. **Truncated** 或 **IoBlocked**（该 root）→ **保守保留** marker（可能仍有未见文件）。
5. 同目录 rename + 仍有未配对文件 → 保留 marker（`probe_partial_dir_keeps_marker`）。

测试：`probe_complete_consumes_dir_marker`（唯一文件全部配对 → 无 `? dir/`）、`probe_truncated_keeps_dir_marker`、`probe_ioblocked_keeps_dir_marker`、`probe_partial_dir_keeps_marker`。

### B.3.4. 读取分类与预算

| 类 | 条件 | 读取与预算 |
|---|---|---|
| **Known-OID exact** | HEAD tree / index stage-0 已记录 OID；两侧 kind 相同且 OID 相同 | 不读对象或 worktree；遵循 Git 内容寻址语义 |
| **Worktree-OID exact** | worktree/symlink 本调用流式计算 Git blob OID，与 HEAD/index Known OID 相同 | 计入 WorktreeReadBudget |
| **Gitlink exact** | 两侧 mode=gitlink 且 commit OID、mode 相同 | 不读取子模块对象；允许本地对象缺失 |
| **Inexact** | 成功读取内容并计算 spanhash；仅 Regular↔Regular | worktree 侧计 WorktreeReadBudget；对象侧计 ObjectReadBudget |

| 预算 | 计入内容 | 上限 | 触顶行为 |
|---|---|---|---|
| **WorktreeReadBudget** | hydrated worktree / LFS 源 / worktree hash/spanhash | 单文件 2MiB；合计 64MiB；4096 tasks；评分批次总 5s；并发 8 | 每路径含零字节先消费 task；触顶/总 deadline 回收在途任务，跳过未完成 edge；warning |
| **ObjectReadBudget** | 仅 inexact 所需的 loose/pack blob 解码 | 单对象 2MiB；合计 64MiB；64 objects；5s | Known-OID exact 不受影响；跳过对应 inexact；warning |

HEAD tree/index 的 OID 本身是 Git 数据模型中的内容寻址标识，足以用于 exact；status 不为此重算整个对象，也不引入 `verified_reads`、fingerprint、storage generation 或 config_kv 热路径。对象损坏由实际需要内容的 inexact 读取、正常 object loader 和 `fsck` 报告。这样避免首次 status 退化为 O(total blob bytes)，并避免 rename 语义依赖 pack cache 内部实现。

```rust
enum BlobEvidence {
    KnownObjectId { oid: ObjectHash },
    ComputedWorktreeThisCall { oid: ObjectHash },
    Unknown,
}
struct BlobRef {
    kind: Regular | Symlink | Gitlink,
    mode: u32,
    size: Option<u64>,
    evidence: BlobEvidence,
}
```

- 已由主扫描确认存在/类型的 rename destination，其后续可选 hash/spanhash/LFS 内容读取标记 `required_for_base_status=false`：`NotFound` 只移除该 rename 候选并保留主扫描基础记录（race warning）；permission/I/O/timeout 产生 worktree warning、跳过相关 edge，保留 `D + ??`，不得升级为 IoBlocked。只有主扫描尚未可靠确定 existence/type/tracked-dirty 的必要读取失败才产生 `IoBlocked`。
- deleted 源产生：`NotFound` → `deleted`；`EACCES`/I/O → `IoBlocked` + warning + dirty + 排除 rename source。测试：`tracked_source_eacces_marks_dirty`、`eacces_fail_closed_no_porcelain_d`。
- 非 UTF-8 路径：`StatusData` 始终保留 `PathBuf/OsString`，不得因默认 rename 让整个 status 失败。R0 若评分器不能无损处理该 basename，只跳过涉及该路径的 rename 候选，保留基础 A/D/M/`??` 和既有 renderer 行为，并产生 `RenamePathEncodingUnsupported` warning；Unix porcelain v1/v2 `-z` 沿既有路径编码输出 raw bytes。R0.5 再使非 UTF-8 候选参与 exact/inexact；`COMPATIBILITY.md` 标 rename detection partial，而非 status 不可用。

## B.4. 当前算法（代码现状，待替换）

`detect_renames_in_changes` 步骤（`status.rs:703–784`）：

1. `head_blobs ← load_head_tree_blobs(head_oid)`；对每个 `new[i]` 算工作区 `calc_file_blob_hash`。
2. 按 `deleted` 迭代顺序贪心：精确匹配 `head_blobs[d]==new_hashes[j]` → score 100；否则 `filename_similarity(basename)`（LCS×100/maxlen）。
3. score ≥ threshold 则配对；更新 `deleted`/`new`/`renamed` 并排序。

**已知缺陷**（R0 必须消除）：错误快照（§B.4.1）、cwd-relative 键、折叠过早、无 RM 合并、静默 hash 失败、无阈值校验。

### B.4.1. 快照与 `BlobRef`（唯一定义）

```rust
struct RenameSnapshot { old_map: HashMap<PathBuf, BlobRef>, new_map: HashMap<PathBuf, BlobRef> }
```

| 侧 | old_map | new_map |
|---|---|---|
| staged | HEAD | index stage-0 |
| unstaged | index stage-0 | worktree |

**exact 合法组合**（`match_pairs` Exact 阶段硬门禁；其它组合静默跳过该候选）：

| old.evidence | new.evidence | 允许 exact？ |
|---|---|---|
| KnownObjectId(a) | KnownObjectId(b) | `a==b` 且 kind 同；gitlink 另要求 mode 同 |
| KnownObjectId(a) | ComputedWorktreeThisCall(b) | `a==b` 且 kind 同 |
| ComputedWorktreeThisCall(a) | KnownObjectId(b) | `a==b` 且 kind 同（对称） |
| 任一端 Unknown | * | 否；仅可在内容成功读取后参与 inexact |

- HEAD/index entry 直接填充 `KnownObjectId`；OID 长度/hash kind 在反序列化边界验证。gitlink 不要求本地 commit 对象存在。
- worktree 成功哈希 → `ComputedWorktreeThisCall`；失败见 §B.6.0.1。
- **size/OID（worktree）**：`worktree_blob_oid_and_size` → `Result<(ObjectHash,u64),_>`（LFS fallible；symlink=`readlink` 字节；预算=WorktreeReadBudget）。
- **对象读取（仅 inexact）**：复用现有 object loader/cache，按 OID 去重；Missing/Corrupt/Unavailable/BudgetExceeded 只跳过依赖该对象的 inexact 候选并产生确定 warning，不改写基础 A/D/M 结果。
- 测试：`known_oid_exact_does_not_read_blob`、`worktree_exact_matches_known_index_oid`、`object_read_budget_skips_only_inexact`、`gitlink_missing_object_exact_by_oid_mode`、`object_budget_independent_of_worktree_budget`。

- **空文件**：exact 仅当两侧 `BlobEvidence` 合法组合且 oid 相同；inexact 跳过 `size==0`。
- worktree 可选 rename 内容读取失败遵循 §B.3.4 上述 `required_for_base_status=false` 规则；基础状态必要读取失败才进入 §B.6.0.1，且禁止把“无法判定”伪造成 D。

### B.4.2. `match_pairs` 算法（Git diffcore 阶段顺序）

1. **Exact**：仅处理 §B.4.1 合法 `BlobEvidence` 组合；桶键=`(oid,kind)`；含 `size==0`；桶内 basename 相等优先 → `old↑`/`new↑` 一对一。
2. `threshold == 60000` → 结束。
3. **Basename**（始终执行；Git `find_basename_matches`）：
   - 仅当某侧 basename 在候选集中**唯一**时尝试配对；重复 basename **不**在此阶段配对（留给 exhaustive 或目录推断 R0.5）。
   - 每对计算 spanhash → `internal_score`；须 `score >= threshold`。
4. **renameLimit 门控**：`limit>0` 且 `sources > limit || destinations > limit`（与 `diff.rs` / Git 一致的 per-side 语义）→ **跳过** exhaustive（basename 已完成）；否则继续。
5. **Exhaustive inexact**：per-dest top-K=4；`max_similarity_comparisons=500_000`（仅 status）。为所有达阈值 edge 生成 `(score, same_basename, old_path_bytes, new_path_bytes)`，按 `score↓, same_basename(true first), old_path_bytes↑, new_path_bytes↑` 全局稳定排序，随后单遍 greedy：source/destination 均未消费才配对。这与当前 `diff.rs` 的确定性 greedy 方向一致；不声明最大权匹配。top-K 对每个 destination 使用同一排序保留最优 4 条；若第 5 条在排序上更优必须替换最差边。**触顶规则**：丢弃整个 exhaustive 阶段已评结果，仅保留 exact+basename + warning。
6. `RenameMatch { exact, internal_score }`；`RenameDetectStats { … }`。测试用具体 2×2/5×1 分数矩阵锁定 source 冲突、same-basename tie 和 path-byte tie。

**退出码优先级**（闭合 `--exit-code` 抢先 `silent_exit(1)` 使顶层 `cli.rs` exit-9 永不运行）：

#### B.6.0.1. `IoBlocked`（EACCES/I/O — **禁止伪造 deleted**）

`EACCES`/I/O 只证明「无法判定」，**不是** `NotFound`。现有 `status_untracked.rs` 将 `symlink_metadata` 错误当删除是缺陷，R0 **必须**修正。

| 输出 | 行为 |
|---|---|
| **human / short / porcelain v1/v2** | **fail-closed**：不输出部分 dirty/porcelain 正文；`CliError::fatal(msg).with_stable_code(StableErrorCode::IoReadFailed)`（**唯一**：`LBR-IO-001`；**禁止**另造 `StatusIoBlocked`/`WorktreePermissionDenied` 枚举值）；exit 属 §B.5 第 1 档 |
| **JSON / API** | 见下方 **io_blocked schema**；`--exit-code-on-warning` → 9 |

**`is_clean` / `--exit-code`（JSON 可输出 partial 时）**：

- `data.io_blocked` **非空** ⇒ `is_clean = false`（「无法判定」**不得**报告为 clean），即使无已确认 rename/D/M/`??`。
- `--exit-code`：`io_blocked` 非空视为 dirty → exit **1**（仍服从 §B.5：on-warning 的 9 ≻ 1；porcelain fail-closed 走 fatal 档）。
- JSON 固定输出 `base_scan_complete` 与 `rename_detection_complete`。前者仅在基础扫描 IoBlocked/timeout 时 false；后者在 probe truncated/IoBlocked、读取 warning、任一 read/similarity/renameLimit budget、非 UTF-8 候选跳过或其它 edge 降级时 false。兼容汇总字段 `complete = base_scan_complete && rename_detection_complete`，不得把降级后的 D+?? 宣称为完整 rename 结果。

测试：`json_io_blocked_is_clean_false`、`exit_code_ioblocked_is_one`（仅不可读、无其它变更 → `--json` `is_clean:false`；`--exit-code` → 1）。

**`data.io_blocked[]` JSON schema（公开契约；`json_io_blocked_schema_snapshot` 锁定）**：

```json
{
  "path": { "display": "<escaped repo-relative>", "raw_base64": null | "<base64 OS bytes>" },
  "staged": "R" | "M" | "A" | "D" | "T" | "U" | null,
  "reason": "permission_denied" | "io_error" | "io_timeout",
  "rename": null | { "from": "...", "to": "...", "score": 0-100 }

```

- `path.display` 使用与非 `-z` porcelain 一致的可读转义；Unix 非 UTF-8 时 `raw_base64` 必填并承载原始 OsStr bytes，UTF-8 时为 null。Windows 使用 WTF-8/平台既有可逆编码 helper，并在 docs 中固定。数组按原始 OS 路径字节/编码单元升序；同 raw path 去重保留首次，禁止按 lossy display 去重。该对象形态是新增 JSON 契约，不提供时而 string、时而 object 的 union。
- 每条 IoBlocked **必须**伴随 `warnings[]` 一项：`worktree_permission_denied`（EACCES）、`worktree_io_timeout`（deadline）或 `worktree_read_failed`（其它 I/O）；`source=worktree`。

### B.6.1. Human / Short / `-z`

- human long：`renamed: <old> -> <new>`
- short 非 `-z`：`R  <q_old> -> <q_new>`（`quote_pathname`，见 §B.6.6）
- **short `-z`**：`XY SP <new_raw> NUL <old_raw> NUL`（rename）。（IoBlocked 在 short/porcelain **不**产出行——见 §B.6.0.1 fail-closed。）
- 非 UTF-8 路径若因 R0 限制未参与 rename，仍按基础 D/A/`??` 的既有 raw-path 行为输出，不得令整个 status 失败。
- 裸 `libra status -z`：`resolution.format.z_explicit && !short && !porcelain && !long` → 强制 porcelain v1 + NUL（§B.4.3 `StatusFormatFlags`）

### B.6.2. 路径投影

| 格式 | 路径基准 |
|---|---|
| porcelain v1/v2、JSON | **repo-relative**（从仓库根；子目录 cwd 不变） |
| human、short | `status.relativePaths=false` → repo-relative；`true`（默认）→ cwd-relative |
| unmerged、ignored | **同上**：`StatusData` 存 repo-relative；human/short 投影 |

`collect_status_data` **禁止**在数据收集末尾对 porcelain/JSON 路径做 cwd-relative；unmerged/ignored 与 rename 同规则。

### B.6.3. Porcelain v1

**非 `-z`**（C-style quote 后空格分隔）：

```text
R  <old_path> -> <new_path>
RM <old_path> -> <new_path>
```

**`-z`**：`XY SP <new_raw_bytes> NUL <old_raw_bytes> NUL`（**原始路径字节**，不 quote；与 Git 一致）。

### B.6.4. Porcelain v2

字段 **恒为** Git 语义：`mH/hH`=HEAD，`mI/hI`=index，`mW`=worktree（[porcelain v2](https://git-scm.com/docs/git-status#_porcelain_format_version_2)）。

**staged rename `xy=R.`** 非 `-z`：

```text
2 R. N... <mH> <mI> <mW> <hH> <hI> R<pct> <new>\t<old>
```

**解析约定（测试 + 实现辅助）**：whitespace split 后固定位：
`[0]=2`，`[1]=xy`，`[2]=sub`，`[3]=mH`，`[4]=mI`，`[5]=mW`，`[6]=hH`，`[7]=hI`，`[8]=R<pct>`，其后为路径（非 `-z` 时 `<new>\t<old>` 可能粘在末 token）。
**禁止**用「首个 `starts_with('R')`」取 score：`xy=R.` 会先于 `R75` 被命中。score 字段须 `fields[8].strip_prefix('R').filter(|s| !s.is_empty() && s.chars().all(|c| c.is_ascii_digit()))`。

**staged `-z`**：`… R<pct> <new_raw> NUL <old_raw> NUL`

**链式 rename**（staged `a→b` + unstaged `b→c`）：**前提** `status.renameUntracked=true`（或测试显式 `libra config status.renameUntracked true`）。默认 `renameUntracked=false` 时第二跳为 `D`+`??`，**无** `.R`。启用后：**两条** `StatusRecord`（`R.` + `.R`），short 两行；JSON 两条 `renames[]`；v2 两行 `2`。测试：`chain_rename_two_records`（**须**设 renameUntracked）；`chain_rename_default_untracked_d_and_question`（默认 `D`+`??`）。

**unstaged `.R`**：**前提** `renameUntracked=true`（所有 `.R`/`unstaged_rename_*` 测试须显式启用）。若记录无 staged 分量，则 `mH/hH` **复制** `mI/hI`（Git fix-up）；链式 `a→b` staged + `b→c` unstaged 时 `.R` 的 `hH==hI==b@index`。

契约测试（逐字段核对 fixture OID/mode，禁止仅前缀/`R100`/路径；禁止接受全零 OID 或默认 mode 回落）：
- `porcelain_v2_staged_rename_mode_hash_fields`：**staged inexact** rename（内容局部修改使 `hH ≠ hI`，且可构造 `mH≠mI` 如 mode 变更）→ 用 fixture 已知值逐字段断言 `mH/mI/mW/hH/hI` 与 `R<pct>`/`paths`；**禁止**仅测 exact/`R100`（无法发现 hH/hI 对调）。
- `porcelain_v2_unstaged_dot_r_hash_fixup`：`renameUntracked=true`；纯 `.R` → `hH==hI==<非零 index OID>`、`mH==mI==<具体 mode>`、`mW`/`路径` 为真实 worktree 值（**禁止**全零回落通过）。

### B.6.5. JSON

顶层 `data.renames[]`：

```json
{ "from": "a.txt", "to": "b.txt", "score": 100, "exact": true, "staged": true, "unstaged": false }
```

- `exact`：OID 相同且 kind 相同（与 score 独立）。
- `score`：`display_score = min(100, internal_score / 600)` **向下取整**（与 `diff.rs:2487–2503`、`diff.rs:2718` 一致）。**允许** `{exact:false, score:100}`（reordered chunks：`internal_score≥60000` 但 OID 不同）。`exact=true` 时 score **亦**为 100。
- 测试拆分（禁止互相排斥）：
  - `json_inexact_reordered_score_100`：`internal=60000` → `{exact:false, score:100}`。
  - `json_inexact_spanhash_score_floor`：**仅**断言 `59999→99`（**不**断言 “non-exact 禁止 100”）。
- `staged`/`unstaged`：该 rename 配对是否来自 staged / unstaged 检测侧（非 XY 字符）；RM 仅 `staged=true`；纯 `.R` 仅 `unstaged=true`。
- `data.warnings[]`：见 §B.5。
- **保留** nested `staged.renamed` / `unstaged.renamed`（`{from,to}` 无 score）。

快照测试：`json_rename_rm_partial_chain` 锁定 RM、partial score、链式两条。

### B.6.6. 路径引用（`core.quotePath`）

`ResolvedStatus.quote_path`（§B.5 级联）传入 `StatusData`；共享 helper `quote_pathname(path, quote_path: bool)` 供 human short、porcelain v1/v2（非 `-z`）使用：

- **始终**转义 TAB、LF、CR、`\`、`"` 等控制/特殊字符（与 `core.quotePath` 无关）。
- `core.quotePath=true`（默认）：另转义 UTF-8 中 `>0x7F` 的非 ASCII 字节为 `\xxx`。
- `core.quotePath=false`：非 ASCII **不**额外转义，但控制字符仍转义。
- `-z`：**不** quote，raw UTF-8 路径字节 + NUL。
- 测试：`porcelain_quote_path_utf8_non_ascii`、`short_quote_path_core_false`、`quote_path_always_escapes_tab`。

## B.7. 大文件与性能

- **WorktreeReadBudget（2 MiB/file、64 MiB total、4096 tasks、5s batch、并发 8）**：inexact 与 worktree hash 读取；零字节也计 task；与 ObjectReadBudget 分离。
- **ObjectReadBudget（单对象 2 MiB / 合计 64 MiB / 64 objs / 5s）**：只覆盖 inexact 所需 blob 内容解码；Known-OID exact 不读取对象。
- **renameLimit**：`sources > limit || destinations > limit` 跳过 exhaustive（与 `diff.rs` / Git 一致）；`limit==0` 不截断；basename 阶段（§B.4.2.3）始终执行。
- **similarity 预算**：status `max_similarity_comparisons=500_000`（**唯一** inexact 边/比较上限；**无**另设 1M edge-cap / `inexact_edges_cap` warning）；diff 默认 `rename_comparison_budget: Option<u32> = None`。配置 `diff.renameComparisonBudget`：非负整数；`0`→无上限；非法 fail-closed。R0-1 扩展 `ResolvedDiffConfig`。
- **ProbeBudget**：`max_enumerated_entries=50k` 与 `max_qualified_destinations=10k` 分别跨 roots 聚合并分别报告触顶原因。
- **I/O no-progress deadline**：所有潜在阻塞读取由可终止 worker 执行；单操作 10s 无进展即回收。仅基础状态必要操作映射 `IoBlocked(io_timeout)`；可选 repo-object inexact 读取映射 Metadata warning，可选 worktree rename 内容读取映射 Worktree warning，并只丢弃相关 edge。无 detached 线程/进程。
- diff 通过 `max_blob_bytes: None` 保持现有语义。
